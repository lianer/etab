## 背景

eTab应用当前使用React和TypeScript显示简单的健康状态页面。前端基于Create React App构建，复杂度较低。新的网格布局主页代表了完整的视觉和功能重构，引入了交互式小组件、拖拽功能和隔离代码执行。本设计文档阐述了如何在保持性能、安全性和可维护性的同时实现所提出的能力。

## 目标 / 非目标

**目标：**
- 将现有的健康状态页面替换为动态网格仪表板
- 实现平滑的拖拽功能，支持网格对齐（100px单元格，20px间距）
- 支持多个预置小组件（日历、时钟、待办事项）并具有可扩展架构
- 支持通过代码输入创建自定义小组件，具有JavaScript/CSS隔离
- 应用一致的现代化样式：白色主题，16px圆角，微妙的阴影/模糊效果
- 确保动画流畅且持续时间≤0.3秒
- 在拖拽操作和多个小组件运行时保持性能
- 保持解决方案的模块化和可测试性

**非目标：**
- 仪表板布局的后端持久化（可后续添加）
- 用户认证或多用户仪表板
- 移动端响应式设计（超出桌面优化范围）
- 高级小组件市场或共享功能
- 用户之间的实时协作

## 技术决策

1. **网格实现**：使用绝对定位配合网格坐标系，而非CSS Grid
   - **原因**：绝对定位为拖拽操作、网格对齐和动态卡片大小调整提供了更精细的控制。CSS Grid的自动放置与显式拖拽定位冲突。
   - **备选方案**：使用`grid-column`/`grid-row`映射的CSS Grid - 对于动态大小调整和对齐更为复杂。

2. **拖拽库选择**：使用`react-draggable`进行拖拽，`react-resizable`进行大小调整
   - **原因**：轻量级、维护良好、TypeScript友好的库，易于与React集成。提供网格对齐和边界约束。
   - **备选方案**：`@dnd-kit` - 功能更强大但更重；`interact.js` - 更底层但需要更多集成工作。

3. **代码隔离**：使用基于iframe的沙箱化执行小组件代码
   - **原因**：iframe提供完整的JavaScript/CSS隔离和安全边界。可以沙箱化脚本和样式，同时通过`postMessage`允许通信。
   - **备选方案**：Web Workers中的`eval` - 复杂的CSP问题；自定义VM沙箱 - 实现风险高。

4. **状态管理**：使用React Context + `useReducer`管理仪表板状态
   - **原因**：仪表板状态（卡片、位置、大小）是层次化的，受益于集中式管理且无需外部依赖。
   - **备选方案**：Zustand/Redux - 对此范围过度设计；本地组件状态 - 过于分散。

5. **组件架构**：
   - `Dashboard` - 管理网格和卡片状态的主要容器
   - `GridBackground` - 视觉网格线/背景
   - `Card` - 包含拖拽/调整大小手柄、样式的包装器
   - `CardRenderer` - 渲染卡片内容（预置或iframe）
   - `CardCreator` - 通过代码/文本添加新卡片的UI
   - 预置卡片：`CalendarCard`、`ClockCard`、`TodoCard`

6. **样式方法**：使用CSS Modules配合设计令牌保证一致性
   - **原因**：作用域样式防止冲突，设计令牌确保一致的间距、颜色、圆角半径。
   - **备选方案**：Tailwind - 当前项目中未使用；styled-components - 增加运行时开销。

7. **动画库**：使用CSS过渡配合`react-spring`处理复杂动画
   - **原因**：CSS过渡用于简单属性（透明度、变换），`react-spring`用于拖拽操作中的弹簧物理效果。
   - **备选方案**：Framer Motion - 类似能力但包体积更大。

## 风险 / 权衡

- **多卡片性能**：每个卡片都有拖拽/调整大小监听器和潜在的iframe开销。
  - **缓解措施**：虚拟化屏幕外卡片，懒加载iframe，使用`React.memo`优化重新渲染。
- **iframe通信复杂性**：`postMessage` API需要谨慎的事件处理。
  - **缓解措施**：创建具有类型安全的小组件-主机通信抽象层。
- **拖拽操作卡顿**：拖拽期间的复杂计算可能导致帧率下降。
  - **缓解措施**：使用`transform`实现平滑移动，节流调整大小计算，避免布局抖动。
- **用户代码执行的安全风险**：自定义小组件中的恶意代码。
  - **缓解措施**：iframe `sandbox`属性配合限制性策略，CSP头部，代码验证。
- **包体积增加**：添加多个库会增加初始加载时间。
  - **缓解措施**：仪表板路由的代码分割，tree-shaking，考虑更轻量的替代方案。
